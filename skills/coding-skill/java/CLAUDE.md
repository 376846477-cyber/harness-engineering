# 项目规范

本项目使用Clean Code规范进行Java开发。

## 自动应用的规范

在编写Java代码时，请自动应用以下编码规范：

1. **可读性** - 代码应易于阅读，名称传递信息，格式一致，注释作为补充
2. **可维护性** - 通过抽象隔离变化，依赖抽象而非实现，高内聚低耦合
3. **可靠性** - 防御式编程，资源正确管理，容错和自愈机制
4. **可测试性** - 依赖注入，高内聚低耦合，结果可观测
5. **高效性** - 选择高效算法，合理使用数据结构，避免不必要的性能浪费
6. **可移植性** - 使用标准API，明确指定字符集，避免平台特定代码
7. **安全性** - 输入校验，敏感数据保护，使用安全函数
8. **命名规范** - 类名大驼峰，方法/变量小驼峰，常量大写下划线
9. **格式规范** - 缩进一致，大括号风格统一
10. **注释规范** - Javadoc文档注释，注释与代码同步

## 详细规范

如需查看特定规范的详细内容，可以使用以下命令：
- `/java-clean-code` - 总览所有特征
- `/java-readability` - 可读性规范
- `/java-maintainability` - 可维护性规范
- `/java-reliability` - 可靠性规范
- `/java-testability` - 可测试性规范
- `/java-performance` - 高效性规范
- `/java-portability` - 可移植性规范
- `/java-naming` - 命名规范
- `/java-formatting` - 格式规范
- `/java-comments` - 注释规范
- `/java-exceptions` - 异常处理规范
- `/java-concurrency` - 并发规范
- `/java-security-input` - 输入安全规范
- `/java-security-crypto` - 加密安全规范
- `/java-security-serialization` - 序列化安全规范

## 代码审查检查清单

完成代码后，检查以下要点：
- [ ] 名称是否清晰传递信息？
- [ ] 是否遵循单一职责原则？
- [ ] 是否有适当的安全防护？
- [ ] 异常是否正确处理？
- [ ] 资源是否正确释放？
- [ ] 代码是否易于测试？
