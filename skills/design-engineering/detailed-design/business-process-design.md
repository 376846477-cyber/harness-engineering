## 状态机设计

### 状态机要素

```
状态机 = (状态集合, 事件集合, 转换函数, 初始状态)

状态集合 S = {待支付, 已支付, 已发货, 已完成, 已取消}
事件集合 E = {支付, 发货, 确认收货, 取消, 超时}
初始状态 s0 = 待支付
转换函数 δ: S × E → S
```

### 状态转换表

| 当前状态 | 事件 | 目标状态 | 动作 |
|----------|------|----------|------|
| 待支付 | 支付成功 | 已支付 | 更新支付时间，发送通知 |
| 待支付 | 取消 | 已取消 | 释放库存 |
| 待支付 | 超时 | 已取消 | 释放库存，发送通知 |
| 已支付 | 发货 | 已发货 | 更新物流信息，发送通知 |
| 已发货 | 确认收货 | 已完成 | 更新完成时间，发送通知 |
| 已发货 | 自动确认 | 已完成 | 更新完成时间 |

### 状态机实现

```typescript
enum OrderStatus {
  PENDING = 'pending',       // 待支付
  PAID = 'paid',             // 已支付
  SHIPPED = 'shipped',       // 已发货
  COMPLETED = 'completed',   // 已完成
  CANCELLED = 'cancelled',   // 已取消
}

enum OrderEvent {
  PAY = 'pay',               // 支付
  SHIP = 'ship',             // 发货
  CONFIRM = 'confirm',       // 确认收货
  CANCEL = 'cancel',         // 取消
  TIMEOUT = 'timeout',       // 超时
}

const stateTransitions: Record<OrderStatus, Record<OrderEvent, OrderStatus>> = {
  [OrderStatus.PENDING]: {
    [OrderEvent.PAY]: OrderStatus.PAID,
    [OrderEvent.CANCEL]: OrderStatus.CANCELLED,
    [OrderEvent.TIMEOUT]: OrderStatus.CANCELLED,
  },
  [OrderStatus.PAID]: {
    [OrderEvent.SHIP]: OrderStatus.SHIPPED,
  },
  [OrderStatus.SHIPPED]: {
    [OrderEvent.CONFIRM]: OrderStatus.COMPLETED,
  },
  [OrderStatus.COMPLETED]: {},
  [OrderStatus.CANCELLED]: {},
};

function canTransition(status: OrderStatus, event: OrderEvent): boolean {
  return stateTransitions[status]?.[event] !== undefined;
}

function transition(status: OrderStatus, event: OrderEvent): OrderStatus {
  if (!canTransition(status, event)) {
    throw new Error(`Invalid transition: ${status} -> ${event}`);
  }
  return stateTransitions[status][event];
}
```

## 异常流程处理

### 异常类型

| 异常类型 | 说明 | 处理策略 |
|----------|------|----------|
| 参数错误 | 请求参数不合法 | 直接返回错误 |
| 业务规则违反 | 违反业务规则 | 返回错误，说明原因 |
| 外部服务失败 | 依赖服务不可用 | 重试或降级 |
| 系统异常 | 系统内部错误 | 记录日志，返回友好错误 |
| 超时 | 操作超时 | 取消或重试 |

### 补偿机制

```
补偿事务模式（Saga）：

正常流程：
  T1: 创建订单 → T2: 扣减库存 → T3: 扣减余额 → T4: 发送通知

失败补偿：
  T3失败 → C2: 恢复库存 → C1: 取消订单

补偿流程：
  1. 记录每个步骤的操作日志
  2. 失败时按逆序执行补偿操作
  3. 补偿操作幂等
```

### 幂等设计

```typescript
// 幂等键设计
interface IdempotentRequest {
  idempotentKey: string;  // 幂等键（唯一）
  // ... 业务参数
}

// 幂等处理流程
async function processOrder(request: IdempotentRequest) {
  // 1. 检查幂等键
  const existing = await checkIdempotentKey(request.idempotentKey);
  if (existing) {
    return existing.result;  // 返回已处理的结果
  }
  
  // 2. 处理业务逻辑
  const result = await doProcessOrder(request);
  
  // 3. 记录幂等键
  await saveIdempotentKey(request.idempotentKey, result);
  
  return result;
}
```
