# Harness工程开发模式 Skills

## 简介

Harness工程开发模式是一套完整的软件工程开发流程管理体系，为OpenCode提供端到端的开发流程支持。涵盖设计、编码、测试、质量、部署全生命周期，帮助开发团队提升代码质量和开发效率。

## Skills组成

### 主Skill

- **harness-dev-flow**: 工程开发模式主控制器，协调各个阶段的执行

### 子Skills

1. **harness-design**: 设计阶段
   - 需求分析
   - 架构设计
   - 接口设计
   - 数据模型设计
   - 设计文档生成

2. **harness-coding**: 编码阶段
   - 代码框架搭建
   - 核心功能实现
   - 单元测试编写
   - 代码评审

3. **harness-testing**: 测试阶段
   - 单元测试
   - 集成测试
   - 端到端测试
   - 性能测试
   - 测试报告生成

4. **harness-quality**: 质量阶段
   - 代码规范检查
   - 静态分析
   - 安全扫描
   - 性能分析
   - 质量门禁

5. **harness-deploy**: 部署阶段
   - 构建流水线
   - 环境部署
   - 发布管理
   - 回滚机制
   - 监控告警

## 快速开始

### 安装

将所有skill文件夹复制到OpenCode的skills目录：

```bash
# Linux/macOS
cp -r harness-* ~/.opencode/skills/

# Windows
xcopy /E /I harness-* %USERPROFILE%\.opencode\skills\
```

## 核心特性

### 1. 全生命周期管理

```
需求分析 → 技术设计 → 代码实现 → 单元测试 → 质量检查 → 集成测试 → 部署发布
```

### 2. 质量左移

- 在开发早期发现和修复问题
- 自动化测试和质量门禁
- 持续集成和持续交付

### 3. 可追溯性

- 需求→设计→代码→测试的全链路追溯
- 完整的变更历史和审计日志

### 4. 自动化程度高

- 自动生成设计文档
- 自动执行测试用例
- 自动质量检查
- 自动生成报告

## 配置说明

### 基础配置

在项目根目录创建 `.harness/config.yaml`：

```yaml
project:
  name: 我的项目
  version: 1.0.0
  language: python

workflow:
  auto_proceed: true
  quality_gate: true
  require_review: true

quality:
  code_coverage: 80
  test_pass_rate: 95
  complexity_limit: 10
```

### 质量门禁配置

```yaml
quality_gates:
  - name: 代码覆盖率
    metric: code_coverage
    threshold: 80
    operator: ">="
    
  - name: 测试通过率
    metric: test_pass_rate
    threshold: 95
    operator: ">="
    
  - name: 高危漏洞数
    metric: high_severity_issues
    threshold: 0
    operator: "=="
```
